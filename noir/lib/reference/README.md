# `ieee754_reference`

Hand-ported, deliberately literal IEEE 754 reference implementations. The
companion to the optimised `ieee754` crate: every function here is the
*ground-truth* side of an equivalence proof against its optimised
counterpart.

## Why

The optimised crate trades gates for opaqueness — binary-search CLZ,
inlined sticky-bit shifts, fused conditional branches. To prove the
optimised path is sound we need a transparent reference written directly
against IEEE 754-2019 §3.4 / §4.3 / §5.4.1 / §6.x / §7.4. That's what
lives here. Performance is *not* a concern; clarity is.

## Layout

```
noir/lib/reference/
|-- Nargo.toml
|-- src/lib.nr             module root
|-- src/add_f32.nr         binary32 ADD reference (round 1, this spike)
|-- lampe/                 generated by `lampe` -- gitignored
```

## Status

Round 1 (2026-05-04 spike): only `add_f32_reference` is implemented.
Smoke tests live at `../reference_tests/` and run via `nargo test`.
Bit-for-bit equivalence against `add_float32_with_rounding` on every
round-1 corpus case has been verified locally. The Lean equivalence
theorem (`add_f32_equivalence`) is at
`proofs/Ieee754/ZkpSparql/Ieee754/Equivalence/AddF32.lean` — currently
`sorry`-stubbed; round 2 closes it. See
`proofs/Ieee754/equivalence-spike-2026-05-04.md` for the methodology.

Round 2+ extensions: f32 sub, f64 add/sub. Out of scope for this spike.

## Running the smoke tests

```sh
cd circuits/noir_IEEE754

```

## Re-running Lampe extraction

`lampe` is *not* installed by default. Install once:

```sh
git clone https://github.com/reilabs/lampe /tmp/lampe-build/lampe
cd /tmp/lampe-build/lampe
cargo install --path . --locked
```

Then, from this package directory:

```sh
cd circuits/noir_IEEE754/noir/lib/reference
rm -rf lampe/
lampe
```

The `lampe/` directory is gitignored — re-extract after any change to
`src/*.nr`.
